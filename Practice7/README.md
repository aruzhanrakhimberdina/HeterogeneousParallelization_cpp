## Контрольные вопросы

### 1. В чём разница между редукцией и сканированием?

Редукция сводит весь массив к одному итоговому значению, например к сумме всех элементов. В результате получается одно число.  
Сканирование вычисляет накопленный результат для каждого элемента массива, поэтому результатом является массив той же длины. Основное отличие состоит в том, что редукция возвращает одно значение, а сканирование возвращает массив.

### 2. Какие типы памяти CUDA используются для оптимизации редукции и сканирования?

Для оптимизации редукции и сканирования чаще всего используется разделяемая память, так как она обеспечивает быстрый доступ для потоков внутри блока. Глобальная память применяется для хранения входных данных и итоговых результатов. Локальная память используется для временных переменных каждого потока.

### 3. Как можно оптимизировать префиксную сумму на GPU?

Префиксную сумму можно оптимизировать, выполняя вычисления поэтапно. Сначала префиксная сумма считается внутри каждого блока с использованием разделяемой памяти. Затем вычисляются суммы блоков и добавляются соответствующие смещения ко всем элементам блоков. Такой подход уменьшает количество обращений к глобальной памяти и повышает производительность.

### 4. Приведите пример задачи, где применяется сканирование.

Сканирование используется в задачах сортировки, например в параллельной сортировке по разрядам. Также префиксная сумма применяется при построении гистограмм, обработке изображений и распределении элементов по индексам в параллельных алгоритмах.
