## Контрольные вопросы

### 1. Какие преимущества предоставляют гибридные вычисления?

Гибридные вычисления дают возможность одновременно использовать ресурсы CPU и GPU. CPU хорошо справляется с управлением программой, последовательными этапами и задачами со сложной логикой, а GPU обеспечивает очень высокую скорость при выполнении большого числа параллельных операций. Благодаря такому распределению нагрузки можно добиться более высокой производительности и уменьшить общее время выполнения программы по сравнению с использованием только одного устройства.

---

### 2. Как минимизировать накладные расходы при передаче данных между CPU и GPU?

Накладные расходы возникают из-за необходимости копирования данных между оперативной памятью CPU и памятью GPU. Чтобы снизить эти затраты, важно уменьшать количество пересылок и передавать данные крупными блоками, а не маленькими частями. Также полезно выполнять как можно больше вычислений на GPU после одной передачи данных. Дополнительно производительность улучшают асинхронные копирования (cudaMemcpyAsync), использование CUDA streams и pinned memory, которые позволяют ускорить обмен данными и частично выполнять вычисления параллельно с передачей.

---

### 3. Какие задачи лучше выполнять на CPU, а какие на GPU?

CPU лучше подходит для задач, где присутствует сложная последовательная логика, большое количество условий и ветвлений, а также управление ходом программы. GPU эффективнее использовать для задач, которые легко распараллеливаются, то есть состоят из множества одинаковых операций над большими массивами данных. К таким задачам относятся обработка векторов и матриц, вычисления в машинном обучении, обработка изображений и моделирование физических процессов.

---

### 4. Как можно улучшить производительность гибридного приложения?

Для повышения производительности гибридного приложения важно правильно распределять работу между CPU и GPU, чтобы оба устройства были максимально загружены и не простаивали. Также следует минимизировать время передачи данных между устройствами и по возможности совмещать вычисления с копированием данных (overlap). Улучшения можно добиться подбором оптимального числа потоков OpenMP и параметров запуска CUDA ядра, таких как размер блоков и количество потоков. Наибольший эффект гибридный подход дает в задачах большого размера, где вычислительная часть значительно превосходит накладные расходы на обмен данными.
